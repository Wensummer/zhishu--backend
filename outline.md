# 智枢 · 全链路智能营销经营策略 v0.3

> 合规国产大模型「选型 + 营销」赋能平台
> 参赛赛道：方向三 · 内部效率提升（AI 赋能 C 端全链路私域运营 / 提升个人岗位生产力）
> v0.3 变更：吸收中转比价导航站（HelpAIO 等）的信任机制设计，强化「透明可核验选型」主线与国央企差异化论证

---

## 0. 定位与基本原则

**一句话定位**：依托天翼云已备案的国产大模型能力池，做一套懂客户、会选型、能经营的「ToB 为主」智能营销赋能平台，把电信的合规大模型 API 能力高效卖出去、用起来、续下去。

**卖什么**：天翼云合规模型池（已备案的主流国产模型，如阿里百炼 / 通义、文心、DeepSeek、智谱等）的 API / MaaS 服务，以及配套的 Agent、行业 MCP 等增值能力。

**电信的角色**：可信渠道 + 智能选型 + 全链路营销经营。赚的是服务、选型与增值的钱，不做转售套利。

**与灰色中转站的切割**：我们不是「合规版中转站」。灰中转的本质是供给侧不合法（逆向渠道、混合号池、随时跑路），靠低价套利生存；我们的本质是供给侧合法（天翼云备案模型池）+ 经营侧专业（选型、服务、客户成功）。市面灰中转生态的真实乱象（跑路、掺渠道、倍率随时跳变、企业无法开票）正是本方案痛点论证的一手素材，也是央企差异化的反面注脚。

**四条红线原则**：
1. 只接已备案的国产模型，不碰境外模型、不从第三方灰中转进货；
2. 数据不出境，全程脱敏、权限控制、日志审计、合规留痕；
3. 不替换电信现有 CRM / 营销系统，作为增量赋能层，通过接口取数与回写；
4. ToB 为主、toC 轻量自助，外呼仅作客户经理辅助触点，频次控制、拒接即停。

## 1. 基础能力准备

### 1.1 合规模型池与选型知识库
接入天翼云已备案的国产模型，持续采集每个模型的：
- 能力分级：按任务类型跑标准评测，得到可核验的质量基准；
- 价格：输入 / 输出 token 单价、档位、缓存折扣；
- 稳定性与性能：可用率、首 token 延迟 TTFT、单 token 输出耗时 TPOT、限流阈值，按日 / 周 / 月滚动统计。

汇成可核验的选型知识库（RAG）。目的是给客户做透明、可核验的智能选型，而不是挑渠道套利。

两点补充：
- **渠道纯度承诺**：所有模型均为备案官方渠道直连，渠道构成可审计、可出具证明。灰中转生态中「Max 分组掺逆向渠道、日志里出现其他模型」的渠道掺假是企业客户的核心顾虑，渠道纯度本身就是可售卖的信任资产；
- **证据链输出**：选型知识库的每条结论（能力分级、价格、稳定性）都保留数据来源与采集时间，供推荐引擎引用为可核验证据，而非只给一个分数。

### 1.2 套餐与计费设计
基于真实成本设计正规分层套餐：ToB（大客户：按量 / 包年）、toC（按量 / 包月 plan）。给出每档的标准价与议价区间，沉淀为定价与套餐知识库，供客户经理与 chatbot 调用。分层定价是正常商业行为，合规。

### 1.3 销售话术蒸馏
整理优秀客户经理的成交对话，按场景蒸馏成可复用的话术模板 / Skill：首次接触、异议处理、选型推荐、议价、续费。供售前售后调用；成功对话持续回流，越用越准。

### 1.4 产品与技术知识库
整理模型能力、API 接入、计费规则、Agent / RAG 全链路优化手段等，构建技术 RAG，供售前售后答疑与客户经理随身调用。

## 2. 售前与客户经理赋能（ToB 为主）

客户经理赋能分两步落地，两步共用同一个选型 / 推荐引擎：第一步是「进门前的静态底稿」，第二步是「对话中的动态修正」。

### 2.1 通话前智能简报（第一阶段 · 必做）
销售联系某个客户前，平台离线或按需生成一份针对该客户的简报，让销售心中有数：
- 使用情况（存量客户）：当前在用的模型 / 套餐、用量趋势、余额与到期时间、是否触发过限流或报错、用量涨跌，来自平台用量数据；
- 推荐选型：基于使用情况与需求，推荐应主推的型号 / 套餐，附理由（更省 / 更稳 / 更配场景）、续费 / 升级 / 扩容建议与报价区间。**每条推荐附可核验证据链**：评分公式分解（如 综合分 = 能力分 × 可用率 × 成本系数）、各项数据来源与采集时间。既回应「电信会不会专挑贵的推」的质疑，也是答辩时讲推荐透明性的差异点；
- 随身话术：针对该客户的开场、卖点、可能的异议与应对；
- 商机判断：该客户当前阶段（该续费 / 可升级 / 可加推 Agent）。

数据来源：用量数据 + CRM + 选型 / 定价 / 话术 / 技术四个知识库；新客户无用量数据时，用企业名检索补画像（业务、行业、AI 可落地的增长点）。

此阶段全程离线，无实时延迟、无 live ASR 风险，跑的是「慢路径」（RAG 选型 + 大模型生成），离线生成可不计较速度、充分保证质量。本身即可独立 demo、独立产生价值，是必做基座。

### 2.2 通话中实时 copilot（第二阶段 · 加分 / demo 英雄）
在 2.1 之上加一层实时辅助：通话时做语音转文字（ASR）→ 意图识别 → 商机分类（高 / 中 / 低意向、需求类型），根据客户当场提出的新需求或异议，动态调整简报里的推荐与话术，实时弹屏给客户经理。通话记录回流话术库与商机库，供复盘。

延迟说明：copilot 模式下 AI 不进入说话回路，建议后台异步生成、1~3 秒弹屏即可，不卡对话。实时回路只做轻量意图识别（小模型），真正的重活（RAG 选型）走慢路径异步跑。

### 2.3 toC：轻量自助
金额小、不配销售。一个 chatbot 完成自助选型 + 购买方式推荐（按量 / 套餐），引导下单。

入口形态采用**四问选型**：用 3~5 个结构化问题（场景、量级、延迟敏感度、预算）让 AI 给出推荐与证据链，复用 2.1 的同一选型引擎。该模式已被同类比价导航站验证为低理解成本的入口，开发成本极低、demo 观感好。

### 2.4 外呼通道与合规
外呼只作辅助、不做营销轰炸，全程合规（频次控制、拒接即停、合规话术）。
- ToB：真人客户经理拨打，AI 走 2.1 / 2.2 在旁辅助，不让机器人直接对客；
- toC / 标准化场景：可用 TTS 语音 bot 做续费提醒、回访、满意度调研、预约确认等单向或简单交互（容忍度高的场景）。

## 3. 高质量服务

### 3.1 统一接入
统一网关，一个 key 调用模型池里所有模型（OpenAI-compatible 封装）。客户不用自己对接多家，这是最实在的客户价值。

### 3.2 多面板
- 甲方老板面板：总量消耗、成本、宏观趋势、各部门用量；
- 甲方个人面板：个人用量与额度；
- C 端用户面板：用量、计费、余额；
- **模型横评简约表格**：一张表看全模型池的价格、可用率、TTFT / TPOT、缓存折扣、适配场景，关键指标尽收眼底，降低客户理解成本，也是客户经理的随身比价工具。

### 3.3 可用性监控页（从运维功能升级为信任产品）
7×24 自动运维，做一个模型池可用性的在线状态页，定位不只是运维透明，而是**对外可售卖的信任资产与售前武器**：
- 分模型可用率、参考缓存率、TTFT / TPOT 的实时值与 7 / 30 天历史趋势；
- 模型晴雨表：模型池整体健康度一眼可见；
- **价格与套餐变更公告制度**：任何调价、上下架提前公示、全程留痕。灰中转「倍率说变就变、无通知改价」是企业客户做预算的最大障碍，公告制度本身就是差异化；
- 故障公告与处理时间线，有公告、有交代。

客户经理售前可直接展示此页：我们敢把数据摊开，灰中转不敢。

### 3.4 配置答疑 chatbot
回答接入、配置、计费、报错等技术问题，接 1.4 的技术 RAG。

## 4. 售后回访 + 增值（Customer Success）

基于用量数据做客户成功：
- 用量分析：识别快到期、额度不足、用量激增或下滑的客户；
- 自动建议：结合选型与定价知识库生成续费 / 升级 / 扩容套餐建议，派给客户经理跟进；
- 增值加推：对满意度高的客户，进一步推电信的 AI Agent、行业 MCP 等高价值服务。

「选对 → 用得稳 → 续费扩容」是这套平台的长期价值，也是电信领导最看重的留存逻辑。

## 5. 管理侧

数据大屏 + 营销 / 客户 / 运营 / 产品管理，统一负责配置、监控、调度、分析与合规管理。

**重点能力**：
- 产品配置：模型 / 套餐的上下架管理，控制哪些模型对外可售、面向哪些客户分层；
- 价格与优惠：各模型 / 套餐的定价、折扣、促销活动、议价区间配置，与 1.2 定价知识库联动；
- 运营分析：客户经理人效、商机漏斗与转化归因、续费率；
- 合规：数据脱敏、权限控制、日志审计、合规留痕。

## 7. 合规与安全（作为亮点，不是免责声明）

只接备案模型、数据不出境、脱敏、权限控制、日志审计、合规留痕。央企背书的合规可信，正是企业客户最缺、市面灰中转给不了的东西，把它当护城河来讲。

**痛点对照表（路演核心论证素材，每条都有灰中转生态的真实案例可引）**：

| 灰中转真实痛点 | 国央企平台对应优势 |
| --- | --- |
| 跑路风险常态化：头部站点持续宕机被定性为软跑路，导航站滚动横幅警示「请勿囤积、贪图大额优惠」 | 央企主体、资金安全，预付无跑路风险 |
| 倍率随时跳变：受上游风控影响倍率从 1.5 跳 2.5，无预算确定性 | 合同锁价 + SLA 保障 + 调价公示制度 |
| 财税合规缺口：个人仅能开国外 Invoice，国内开票限对公且收 6% 手续费，起开门槛 200~5000 不等 | 正规增值税发票、对公结算，天然适配企业采购与预算流程 |
| 渠道掺假：官方渠道掺逆向、日志中出现非承诺模型，供给侧本身不合法 | 只接备案模型，渠道纯度可审计、可出证明 |
| 服务上限低：OneMan 站长「需要睡觉，可用率受影响」，客服找不到人 | 7×24 企业级服务 + 客户经理体系 |
| 数据隐私无保障：QQ 邮箱注册、退款索要个人信息 | 数据不出境、脱敏、等保级权限控制与审计留痕 |

## 8. 指标体系（ToB 口径）

线索质量、商机转化周期、客单价、**客户经理人效**（方向三的标准答案）、续费 / 扩容率、单客获取成本。比赛时用「现状基线 vs 目标」对比，数字按真实情况填。

呼应「透明可核验」主线，补充两个指标：
- **推荐采纳率**：简报推荐被客户经理采用 / 被客户接受的比例，直接衡量选型引擎质量；
- **选型相关客诉率**：因推荐不当或价格不透明引发的客诉占比，衡量信任工程成效。

## 9. 比赛落地

**必做主线（保证可靠、能跑）**：通话前智能简报（2.1），即「客户用量 / 需求 → AI 选型 → 生成简报（推荐型号 + 话术 + 商机判断）」。全程离线、不赌实时，用自建 Dify 编排端到端跑通，作为路演 demo 的稳定基座。

**加分项（demo 英雄）**：通话中 copilot（2.2），实时 ASR + 意图识别 + 动态弹屏，复用同一选型引擎。时间紧时实时部分可半 mock 演示。

其余模块用 mock / PPT 带过。
